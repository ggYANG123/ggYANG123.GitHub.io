Git配置公钥和密钥

生成公钥密钥

1.下载 Git

2.生成密钥：

. -t rsa 指定密钥类型为RSA。
. -b 4096 指定密钥长度为4096位。
. -C 后面跟上你的邮箱，用于标识这个密钥。
运行这个命令后，系统会提示你输入一个文件路径来保存密钥（例如/home/user/.ssh/id_rsa），以及两次输入密码（可以为空，如果你不想为密钥设置密码）。

说明：
生成位置一般在C:\Users\user/.ssh/id_rsa
d_rsa # 私钥
d_rsa.pub # 公钥，你可以在本地通过文本文件打开它获取公钥内容

3.添加SSH密钥到远程仓库

GitHub:
登录到你的GitHub账户。

点击右上角的头像，选择“Settings”。

在左侧菜单中选择“SSH and GPG keys”。

点击“New SSH key”按钮。

粘贴你的公钥到“Key”文本框中。

输入一个标题（例如“Work laptop”），然后点击“Add SSH key”按钮

4.测试SSH连接

配置完SSH密钥后，你可以测试SSH连接是否成功：

如果一切配置正确，你会看到一条欢迎消息，表明SSH连接成功。

关于将程序置于后台

使用第三方工具 nssm (Non-Sucking Service Manager)

下载并安装 NSSM:

前往 NSSM 官方网站 下载适合你系统的版本。
解压后将 nssm.exe 放置在一个方便的位置，比如你的项目目录下的某个文件夹内。

创建服务:

打开命令提示符，导航到包含 nssm.exe 的目录。
使用以下命令创建一个新的服务：cmd

在弹出的 GUI 中填写必要的信息：

  • Path: Python 解释器的路径，例如 C:\path\to\venv\Scripts\python.exe
  • Startup directory: 你的 Django 项目的根目录
  • Arguments: manage.py runserver 192.168.200.17:8000
  • 点击“Install service”按钮完成安装。

管理服务:

你可以通过 services.msc 或者使用 nssm start DjangoDevServer, nssm stop DjangoDevServer, nssm restart DjangoDevServer 来管理你的服务。
